## Racoin
Lien du projet : [https://gitlab.univ-lorraine.fr/sauder2/racoin](https://gitlab.univ-lorraine.fr/sauder2/racoin)

Vous avez récupéré le projet Racoin ! C’est le début des problèmes donc...

### Notes préliminaires
Créez un fichier `notes.md` pour y noter toutes vos observations.
Il doit contenir une fiche d’identification du projet, ainsi que les éventuelles actions réalisées avec leurs explications.

Le but n’est pas d’avoir un document administratif, mais un contexte pour vous et les autres personnes travaillant sur le projet.

### Première étape - L’analyse
- [ ] Trouver le ou les langages utilisés
- [ ] Trouver le ou les frameworks principaux utilisés
- [ ] Trouver le but général de l’application
- [ ] Faire une première estimation de ce qu’il faut pour faire marcher l’application

### Deuxième étape - La prise en main
- [ ] Faire fonctionner l’application
- [ ] Créer un mode d’emploi pour la mise en marche (un `docker-compose` par exemple ?)
- [ ] Identifier les dépendances non maintenues

### Troisième étape - La maintenance
- [ ] Mettre à jour les versions des langages et frameworks
- [ ] Noter dans une TODO list les améliorations envisagées
- [ ] Pour chaque idée, estimez de 1 à 10 le temps de modification envisagé, et l’impact attendu (2 notes donc)

### Quatrième étape - Vous êtes si rapide ?
- [ ] Lister les améliorations que vous avez effectuées
